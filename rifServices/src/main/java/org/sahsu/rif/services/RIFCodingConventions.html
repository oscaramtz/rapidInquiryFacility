General Coding Conventions in the RIF Code Base

Package Naming Conventions

Convention Meaning Example
*.businessConceptLayer Contains all the classes that describe domain concepts which would have meaning for the epidemiologists, GIS experts, statisticians and database managers. The properties of the business classes should be understandable to a domain expert who is not involved with coding for the project. rifServices.businessConceptLayer
*.presentationLayer Contains all the GUI code used to create Java-swing based desktop front end applications. It should
  • only access the database indirectly by making calls to RIF service objects
  • support as much guided-data entry as possible, but it should not contain bespoke validation code

We want this layer to contain "dumb" desktop client applications that can later be ported to web applications.

rifStudySubmissionTool.presentationLayer
*.dataStorageLayer Contains all the code in a sub-project that is used in the middleware. It will contain:
  • RIF service definitions (ending in "API"), which publish behaviour used by client applications
  • contain all the code used to construct or execute SQL queries
  • will contain all vendor-specific code that is used to support PostgreSQL and SQLServer.
rifStudySubmissionTool.dataStorageLayer
*.system Contains files which relate to RIF-specific ideas and are used throughout the sub-package. Contains all the code in a sub-project that is used in the middleware. It will contain:
  • RIF service definitions (ending in "API"), which publish behaviour used by client applications
  • contain all the code used to construct or execute SQL queries
  • will contain all vendor-specific code that is used to support PostgreSQL and SQLServer.
rifServices.system

Source File Naming Conventions

Convention Meaning Example
Abstract* Class cannot be instantiated AbstractStudy
Production* Describes a class which is meant to appear in the production version of the software. ProductionRIFStudySubmissionService
Test* Has one of two meanings:
  • It is a JUnit test suite dedicated to a business concept
  • It is a class that should only be used in testing scenarios and not be used for production support
  • TestRIFStudySubmissionService
  • TestGeography
*Interface Describes a Java interface that will be used by a family of classes that support different implementations. ProductionRIFStudySubmissionService
*API Describes a Java interface publishes methods of a RIF service that will be used by other applications. For example, the desktop version of the Study Submission Tool will make use of the RIF service RIFStudySubmissionAPI. rifServices.dataStorageLayer.ProductionRIFStudySubmissionService
